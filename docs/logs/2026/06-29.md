# Development Log - 2026-06-29

### 2026-06-29 (plan 448 完成 — CRUD loadAction + Picker 重构落地)

- **范围**：完成 `docs/plans/448-crud-loadaction-picker-refactor-plan.md` 的全部 5 个 Phase（CRUD `loadAction` 数据获取层 + Picker Dialog 内嵌 CrudRenderer + labelResolveAction/label 缓存 + 文档同步）。
- **Phase 1/2（CRUD loadAction）**：执行前发现 Phase 2 标 `in progress` 但所有 item 已勾选——经 live test 核对发现真实遗留缺陷并修复：
  - **bug A**：`crud-renderer.tsx` 中 `pagination.enabled` 在 `loadAction` + 非 `loadAllData` 时被错误设为 `false`，导致服务端分页模式下翻页按钮消失。修复方式：为 table 引入 `serverPaged` + `pagination.total` 语义（`schemas.ts` / `use-table-pagination.ts` / `table-renderer.tsx`），CRUD 在服务端分页时设 `serverPaged: true` + 传入 server `total`，table 显示分页栏但跳过客户端切片、用 server total 计算 totalPages。
  - **bug B**：`clientSideQueryFiltering` 门控写成 `useLoadAction || loadDataOnce`，导致 plain `source` 模式（带 queryForm）的客户端筛选失效（`crud-query-and-pagination` 3 个测试挂红）。修正为 `!useLoadAction || loadAllData || loadDataOnce`。
  - **bug C**：`paginationTotal`（`summary.total`）误用 `filteredRows.length`，使 `$crud.total` 在 source/data-source 模式显示为当前页行数而非数据集总数。回归到 `resolvedSource.total`（与原始 baseline 一致）。
- **Phase 3/4（Picker）**：前序 session 已在 working tree 落地大部分实现，但存在架构缺陷导致 6/9 测试挂红：
  - **缺陷 1**：直接 `<CrudRenderer>` 实例化 + `regions: {}`，使 `footerToolbar`/`queryForm` 等 schema fragment 未编译就被当作 React child → "Objects are not valid as a React child" 硬崩溃。改为 `props.helpers.render(pickerCrudSchema, { pathSuffix: 'pickerCrud' })` 走 fragment pipeline，region 正确编译。
  - **缺陷 2**：Hooks 顺序违规（`pickerCrudNodeScope` 的 `useMemo` 在 `if (!visible) return null` 之后）。已移除。
  - **缺陷 3**：静态 `options` 模式被错误路由进 CrudRenderer（破坏原 `<ul><li>` 列表 UI 的 `getByLabelText` 交互）。按 plan 原意恢复：仅 `loadAction` 存在时嵌入 CrudRenderer，静态 options 保留列表 UI。
  - **缺陷 4**：`autoFill: { ownerName: '${row.title}' }` 作为普通 `prop` 在渲染期被当公式求值，但 `row` 不可用 → 报错。改为 `lazyEval: true, params: ['row']`，由 Picker 在选中时用 `helpers.evaluateCompiled` 在 row scope 求值。
  - **缺陷 5**：CRUD mode 下 confirm 时取不到已加载行（`options` 为空、`selectionRows` 无缓存），导致 value/label/autoFill 全空。新增 CrudSchema `dataStatePath`（CRUD 把当前 rows 发布到 scope），Picker 在 confirm 时从该 path 读取行数据用于 label 缓存 + autoFill。
  - **lint 修复**：`useCrudLoadAction` effect 的 `eslint-disable react-hooks/exhaustive-deps` 触发 react-compiler 报错；改为补全 deps + 把同步 `setLoading/setError` 移进 async IIFE 满足 `react-hooks/set-state-in-effect`；清理 dead code `latestStateRef`。
- **验证**：focused 单测全绿——`crud-loadaction`(6) + `crud-query-and-pagination`/`data-crud-request-owned`/`crud-binding-and-status`/`data-table-pagination-selection`(30) + `picker-renderer`(9) 全过；`pnpm --filter flux-renderers-data --filter flux-renderers-form-advanced typecheck/build/lint` 全过（lint 0 error）。
- **文档**：`docs/components/crud/design.md` 同步 §3 非目标修订（loadAction 定位）+ §4.1.1/§4.4/§6.1/§7.2 + §8 迁移映射（`api → loadAction`/`loadDataOnce → loadAllData`）；`docs/architecture/data-domain-owner.md` 补 loadAction 说明。
- Key decision：`serverPaged` 是 table 层缺失的语义——服务端分页时 table 显示分页栏（用 server total）但跳过客户端切片；这同时服务 CRUD loadAction 与未来 Picker 场景，不是 picker 专属 hack。
