# Manual Test Issues — 2026-05-19

## 2. Performance Table Full Stress 在 StrictMode host mutations 后出现 `$slot.record` 渲染错误

- **所属模块**: playground / performance-table / table row scope cache / scoped slot region / React StrictMode
- **问题描述**: 在 `#/performance-table` 页面切换到 `Full Stress` 后，执行 `Run 20 Host Mutations` 时，表格 cell 与 expanded row 中的 `${$slot.record.*}` / `${$slot.record.children}` 表达式可能大量报错；happy-dom 回归中 `console.error` 捕获到 312 次 render error。
- **预期行为**: Full Stress 模式在 StrictMode 下完成 host mutations 后仍应保留当前行 `$slot.record` 作用域，页面无 `console.error` / `pageerror`。
- **发现方式**: 回归验证 / 探索性测试
- **状态**: 已修复（`packages/flux-renderers-data/src/table-renderer.tsx` 的 table owner key 已纳入 `RendererRuntime.runtimeId`，避免 keyed `SchemaRenderer` remount 后复用旧 runtime 的 row-scope cache；新增 `data-table-row-scope-identity.test.tsx` keyed StrictMode remount 覆盖，并通过 performance-table happy-dom 与 Playwright focused 回归）

## 3. Collapsible fieldset 没有显示展开/收缩图标

- **所属模块**: flux-renderers-form / fieldset renderer
- **问题描述**: 当 fieldset 设置 `collapsible: true` 时，标题栏（`<legend>`）只渲染了纯文本，没有展开/收缩的方向图标（如 ChevronRight/ChevronDown）。用户无法从视觉上辨别该 fieldset 是否可折叠，也无法直观看到当前展开/折叠状态。相关代码在 `packages/flux-renderers-form/src/renderers/fieldset.tsx:52-65`，`<legend>` 内仅输出 `{title}`。
- **预期行为**: 可折叠 fieldset 的标题区域应包含一个方向图标（展开时朝下，折叠时朝右），点击图标或标题文字均可切换状态，与 `data-collapsed` 属性保持同步。
- **发现方式**: 人工测试
- **状态**: 已修复（`packages/flux-renderers-form/src/renderers/fieldset.tsx` 的 `<legend>` 现在在 `collapsible: true` 时渲染 `ChevronRightIcon`（折叠态）/ `ChevronDownIcon`（展开态），使用 `data-slot="fieldset-collapse-icon"` 标记；`fieldset-interaction.test.tsx` 新增图标存在性、图标切换和非 collapsible 无图标回归）

## 4. ConditionBuilder 分组删除按钮与条目删除按钮样式不一致

- **所属模块**: flux-renderers-form-advanced / condition-builder
- **问题描述**: 分组（group）的删除按钮使用 `variant="outline"` + 文本字符 `×` + `absolute` 浮动定位在卡片右上角，呈现为圆形 X 按钮；而单个条目（item）的删除按钮使用 `variant="ghost"` + `Trash2Icon` + `opacity` 渐显。两者风格不统一，圆形 X 按钮不美观。此外删除按钮未垂直居中、未靠右对齐。
- **预期行为**: 分组和条目的删除按钮统一使用 `Trash2Icon` + `variant="ghost"` 风格；删除按钮垂直居中、靠右对齐（`ml-auto`）。分组删除按钮放入 AND/OR 标题行右侧，条目删除按钮靠右显示。
- **发现方式**: 人工测试
- **状态**: 已修复（`condition-group.tsx` 分组删除按钮从 `absolute` 浮动改为嵌入标题行 `ml-auto` 靠右；`condition-item.tsx` 条目删除按钮添加 `ml-auto`；统一使用 `Trash2Icon` + `variant="ghost"` + `hover:text-destructive`）

## 5. Table 可排序列排序图标不区分方向

- **所属模块**: flux-renderers-data / table-header-row
- **问题描述**: 可排序列无论正序（asc）、逆序（desc）、未排序，都显示同一个 `ArrowUpDownIcon` ↕，仅靠颜色区分是否激活（`text-primary` vs `text-muted-foreground`）。用户无法直观看出当前排序方向，三个状态视觉差异不足。
- **预期行为**: 正序显示 `ArrowUpIcon` ↑（`text-primary`）；逆序显示 `ArrowDownIcon` ↓（`text-primary`）；未排序显示 `ArrowUpDownIcon` ↕（`text-muted-foreground/40` 极淡提示可排序）。
- **发现方式**: 人工测试
- **状态**: 已修复（`table-header-row.tsx:160-168` 根据 `currentSort` 三态渲染不同图标：`ArrowUpIcon` / `ArrowDownIcon` / `ArrowUpDownIcon`，未排序态改为 `text-muted-foreground/40` 极淡样式）

## 6. ArrayField 删除按钮纯图标在卡片底部显得单薄

- **发现方式**: 回归验证 / 设计审查
- **状态**: 已修复（`array-field.tsx` 恢复为 `ghost` + `{t('flux.form.remove')}` 文字按钮，移除 `Trash2Icon` import；`array-editor.tsx` 保持 `ghost` + `Trash2Icon` 行内图标不变）
