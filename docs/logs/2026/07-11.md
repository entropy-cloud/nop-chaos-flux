### 2026-07-11 (bugfix: CRUD listMode cards 首次加载空白 — carrier key 未包含数据版本)

**问题**：CRUD `listMode: "cards"` 模式下，首次切换到卡片标签页时内容为空，点击"下一页"分页后才显示卡片内容。

**根因**：`packages/flux-renderers-data/src/crud-renderer.tsx` 中 carrier wrapper 的 `key` 仅依赖 `paginationState.currentPage + pageSize`。React Compiler 自动记忆化（auto-memoize）了 carrier schema 的创建。当 `loadAction` 异步返回数据后（`rows` 从 0 变为 36），由于 pagination 未变（仍为 page 1 / size 10），key 不变，React 不卸载/重挂载 carrier 子树。记忆化的 schema 仍持有 `items: []`，导致 CardsRenderer 渲染空态。点击"下一页"使 key 变化（page 1→2），强制重挂载，此时新 schema 的 `items` 正确，卡片内容出现。

**改动**：

- `packages/flux-renderers-data/src/crud-renderer.tsx:656` — carrier key 增加 `filteredRows.length` 段，使数据加载（0→N）同样触发 carrier 重挂载
- 更新注释说明设计背景

**验证**：

- flux-renderers-data 单测 656/656 ✅

- typecheck 55/55 ✅ / build 29/29 ✅

### 2026-07-11 (bugfix: CRUD listActions 在 toolbarLayout 中重复渲染)

**问题**：`standard-crud` 页面中，当 schema 同时定义了顶层 `listActions` 数组和 `toolbarLayout.header: ["listActions", "pagination"]` 时，批量删除按钮被渲染了两次——一次内联在 toolbar 旁边（正确），一次在 toolbarLayout blocks 行与分页器一起（重复）。同时 `toolbarLayout.footer: ["statistics", "switch-per-page"]` 与 `footerToolbar` 共存导致底部"共 X 条"重复出现。

**根因**：

- `crud-renderer.tsx` 的 header 区域无条件内联渲染 listActions（`crud-list-actions` slot），同时 `CrudToolbarBlocks` 在 `toolbarLayout.header` 含 `"listActions"` 块时又渲染了一遍相同内容，造成重复。旧测试 `data-crud-rendering.test.tsx:210` 甚至将此 bug 固化为 `toHaveLength(2)`。
- `standard-crud.json` 的 `footerToolbar`（"共 X 条 · 选中 X 条"）与 `toolbarLayout.footer` 的 `"statistics"` 块（"共 X 条"）语义重叠。
- CRUD 通过 `toolbarLayout` 渲染 `pagination`/`switch-per-page` 块时，内部 `TableRenderer` 的内置 `TablePaginationBar`（含分页 + 每页条数选择 + X-Y of Z 统计）仍然渲染，导致分页器和每页条数选择器各有两份。

**改动**：

- `packages/flux-renderers-data/src/crud-renderer.tsx` — `resolveToolbarBlocks` 新增过滤：当 `hasListActions` 为 true 时，从 header/footer blocks 中移除 `"listActions"` 类型块（listActions 始终由内联 slot 渲染，blocks 不应重复）；新增 `hasExternalPaginationControl` 计算：当 `toolbarLayout` header/footer 含 `"pagination"` 或 `"switch-per-page"` 块时，向内部 table schema 传入 `pagination.hideBar: true`
- `packages/flux-renderers-data/src/schemas.ts` — `TableSchema.pagination` 新增 `hideBar?: boolean`：为 true 时 table 保留分页逻辑（行切片等）但不渲染 `TablePaginationBar`
- `packages/flux-renderers-data/src/table-renderer.tsx` — `TablePaginationBar` 渲染条件新增 `!schemaProps.pagination?.hideBar`
- `apps/playground/src/complex-pages/page-schemas/standard-crud.json` — `toolbarLayout.footer` 从 `["statistics", "switch-per-page"]` 改为 `["switch-per-page"]`（footerToolbar 已提供总数+选中数，移除重复的 statistics 块）
- `packages/flux-renderers-data/src/__tests__/data-crud-rendering.test.tsx` — Bulk Delete 按钮从 `toHaveLength(2)` 改为 `toHaveLength(1)`，移除对 `header-toolbar-list-actions` slot 的断言
