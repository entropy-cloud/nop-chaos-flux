# Manual Test Issues — 2026-05-11

## 1. tabs with form 切换后输入值丢失

- **所属模块**: playground / tabs lab / form
- **问题描述**: 在 `http://localhost:5173/#/lab/tabs` 的 `Tabs with forms` 场景中，先在一个 tab 内输入表单值，再切换到另一个 tab 后切回，之前输入的内容会丢失，看起来像表单草稿被重置了。
- **预期行为**: tabs 只切换可见面板，不应重置 tab 内 form 的本地值；切回原 tab 后应看到之前输入的内容仍然保留。
- **发现方式**: 人工测试
- **状态**: 已修复（`packages/ui/src/components/ui/tabs.tsx` 默认保持 inactive panel mounted；`packages/flux-renderers-basic/src/__tests__/basic-page-layout-structure.test.tsx` 补充 tabs 内 form 值保留回归测试）

## 2. flux basic 表格 Inspect 按钮点击后未弹出对话框

- **所属模块**: playground / flux basic / table row action
- **问题描述**: 在 `#/flux-basic` 页面中点击用户表格行内的 `Inspect` 按钮后，没有弹出 `User Details` 对话框，行级详情内容也没有显示。
- **预期行为**: 点击 `Inspect` 后应打开对话框，并显示当前行用户的用户名与邮箱。
- **发现方式**: 人工测试
- **状态**: 已修复（`apps/playground/src/pages/fluxBasicPageSchema.json` 改为使用表格行作用域约定的 `$slot.record.*`；`apps/playground/src/pages/flux-basic-page.debugger.test.tsx` 增加行操作打开对话框回归测试）

## 3. flux basic 的 Submit only / Array child items 删除按钮无效

- **所属模块**: playground / flux basic / array-editor
- **问题描述**: 在 `#/flux-basic` 页面 `Submit only / Array child items` 场景中，点击 reviewer 行的删除按钮后，当前条目没有被移除，界面与表单值都保持不变。
- **预期行为**: 删除按钮应移除对应 reviewer 条目，并同步更新界面与提交值。
- **发现方式**: 人工测试
- **状态**: 已修复（`packages/flux-renderers-form-advanced/src/array-editor.tsx` 在调用 `form.removeValue()` 前先更新 `itemsRef`，避免删除过程把旧数组重新写回；`apps/playground/src/pages/flux-basic-page.debugger.test.tsx` 与 `packages/flux-renderers-form-advanced/src/__tests__/{form-array-validation.test.tsx,composite-item-id.test.tsx}` 增加/校正回归覆盖）
