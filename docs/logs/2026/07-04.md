# Development Log - 2026-07-04

### 2026-07-04

- **flux-guide 类型生成 + markdown JSON 校验基础设施**：实现从 `RendererDefinition` 源码自动生成 TypeScript 类型定义 (`schema.d.ts`) 和校验 markdown JSON 示例的两个脚本。
  - **背景**：`flux-types/schema.d.ts` 此前纯手工维护，易漂移；markdown 示例无自动校验。
  - **方案**：解析源码中的 `RendererDefinition` 对象（`type`/`fields`/`propContracts`/`eventContracts`），生成接口 + 校验 markdown 块。两个脚本都用纯 Node，无构建步骤。

- **`flux-guide/scripts/shared.mjs`** — 通用工具层：
  - `matchBracket(text, start)`：修复了不处理 `//` 和 `/* */` 注释的 bug（源文件中注释包含反引号 `backtick` 导致括号失配，`basic-renderer-definitions.ts` 的 page/container/button 等基础类型全部被跳过）。
  - `jsoncParse(text)`：重写为状态机 `jsoncToJSON()`，支持单引号、未引号属性名 (`key` → `"key"`)、尾逗号（含逗号后空格换行再闭合）、`//` 和 `/* */` 注释。
  - `extractDefinitions(filePath)`：正则从 `\w+(RendererDefinitions|RendererDefinition|Definitions|Definition)` 数组/对象中提取定义（新增 `Definitions` 后缀支持，修复了 `w2aDataCompositionDefinitions` 未被识别的问题）。
  - `KNOWN_DEFINITION_FILES`：从 12 个文件扩展到 29 个文件（新增 17 个 form-advanced 定义文件）。

- **`flux-guide/scripts/generate-types.mjs`** — 类型生成：
  - 读取 29 个定义源文件 → 输出 `schema.d.ts` + `index.ts`。
  - 当前加载 92 个 renderer 定义（原 58 个）。类型校验 `tsc -p flux-guide/tsconfig.json` 通过。
  - 修复：`input-month/input-quarter/input-year` 从 `InputPeriodSchema` 重命名为 `InputMonthSchema`/`InputQuarterSchema`/`InputYearSchema`（重复接口名导致类型错误）。
  - 修复：`index.ts` 自动生成逻辑改为完整重写（原逻辑错误地保留了不完整的 import 语句导致 TS1003 `Identifier expected`）。

- **`flux-guide/scripts/validate-json.mjs`** — markdown JSON 校验（自定义）：
  - 扫掠 `flux-guide/**/*.md` 的 ` ```json`/` ```jsonc ` 块，解析后校验字段是否在定义中。
  - 移除 `glob` 包依赖，改为递归 `readdirSync`。
  - 修复 `if (node.__proto__) return;` bug（所有 JSON.parse 对象都继承 `Object.prototype`，`__proto__` 始终为 truthy，导致 0 nodes checked）。
  - 结果：342 nodes checked，0 errors，196 warnings（warnings 来自 runtime 继承字段如 data-source 的 `action`/`args`/`name` 等，这些在 schema type 定义中但不在 `RendererDefinition.fields` 中）。

- **`flux-guide/scripts/validate-with-compiler.mjs`** — markdown JSON 校验（基于 `@nop-chaos/flux-compiler.dist`）：
  - 直接调用 `flux-compiler` 的 `validateSchema()`，获得最准确的校验（含 region、action selector、dollar reference 等）。
  - 通过相对路径 `import()` 从 `packages/flux-compiler/dist/index.js` 加载，无 tsx 依赖。
  - 结果：102 nodes checked，12 errors（均为 预期 runtime 特性无法静态校验：`body` 字符串简写、CRUD toolbar `statistics`、builtin action `confirm`/`alert`、scope injection `$crud`），74 warnings。

- 关键记录：
  - RendererDefinition 的 `fields` 只包含该 renderer 特有的字段，不含继承自 schema type 的字段（如 data-source 的 `action`/`args`/`name`）。`validate-json.mjs` 因此对继承字段报 Unknown key，`validate-with-compiler.mjs` 同样报 Unknown property。这是定义模式本身的特性，非 bug。
  - `packages/flux-core/src/constants.ts` 的 `BUILT_IN_ACTION_REGISTRY` 不含 `confirm`/`alert` ——它们有 ActionSchema 类型和 runtime dispatch，但不在 compiler 的 builtin action 注册表中。
