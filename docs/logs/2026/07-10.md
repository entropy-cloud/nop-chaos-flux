# Development Log - 2026-07-10

### 2026-07-10 (文档: 补充 Layout Renderer 选型决策与 className 路由)

**背景**：用户反馈 container 的 `className: "flex flex-wrap"` 不生效，暴露了架构文档中 layout renderer 选型决策和 `className` 路由说明的缺口。

**分析发现**（通过源码阅读 `container.tsx`）：

- Container 是**双层 DOM**：外层 `<div class="nop-container">` + 内层 `<div data-slot="container-body">`
- `className` 挂在外层，不影响子节点排列
- body 的 flex 布局由 semantic prop（`direction`/`wrap`/`gap`/`align`）或 `bodyClassName` 控制
- `type: "flex"` 是**单层 DOM**，`className` 直接控制根节点，无需了解双层路由

**文档更新**：

1. **`docs/architecture/styling-system.md`** — 新增 `§4.4 Layout Renderer 选型决策`：
   - Container / Flex / Grid 三者的选型对比表
   - Container DOM 结构图 + className 路由说明（附 HTML 示例）
   - 明确指南：简单 flex 用 `type: "flex"`，已有 container 用 semantic prop 或 `bodyClassName`

2. **`docs/components/container/design.md`** — §12 更新：
   - 添加完整 container DOM 结构图（header/body/footer 三层）
   - 明确标注 className → 外层、bodyClassName → data-slot、semantic prop → 内层 body
   - 禁止性指南：不要用 className 给 body 加 flex

3. **`docs/components/flex/design.md`** — §11 补充：
   - flex 与 container 的关键差异：单层 vs 双层 DOM

**dashboard JSON** 已使用正确的 `type: "flex"` + `direction: "row"` + `wrap: true` + `gap: 12` 方案。stat 卡片宽屏 6 列一行、窄屏自动换行。

- **验证**：typecheck ✅ · 110/110 单测 ✅
