### 2026-04-15

- Created Plan 91 (`docs/plans/91-form-field-state-normalization-refactor-plan.md`): refactor form state storage from multiple separate Boolean Maps to a single flat `fieldStates: Record<string, FieldState>` Map, following React/Redux official best practices for state normalization.
- Analysis context: compared three designs (current multi-map, flat single-map, nested tree structure) against React's "Choosing the State Structure" guidelines and Redux's "Normalizing State Shape" recommendations; concluded that nested structures are unsuitable for Zustand immutable semantics due to O(d) deep spread overhead.
- Identified code duplication hotspots: `validationErrorsEqual` duplicated in `form-store.ts` and `form-runtime-status.ts`; `createPrefixedStore`/`createItemStore` share ~80 lines; `createPrefixedFormProxy`/`createItemFormProxy` share ~100 lines; data-attribute setting repeated in 9+ renderer files.
- Key decision: single flat `FieldState` map is the optimal balance for React/Zustand — reduces path string duplication, simplifies array remapping to single Map traversal, and eliminates code repetition.
- Plan phases: (1) type definition refactor, (2) store implementation refactor, (3) array remapping simplification, (4) React hooks adaptation, (5) subform projection extraction, (6) renderer cleanup, (7) documentation update.
