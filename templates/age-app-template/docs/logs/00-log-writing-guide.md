# Development Log Writing Guide

## Purpose

Daily logs preserve short implementation memory that should survive chat resets.

## Rules

- one file per day: `docs/logs/YYYY/MM-DD.md`
- append new entries at the top for the same day
- keep entries short and factual
- link to plans, docs, or code paths when useful

## Typical Entry Content

- what landed
- what was verified
- what docs were updated
- what remains next
