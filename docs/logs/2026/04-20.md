  - ErrorBoundary for render error handling

  - Missing ScopeFactory in RenderEnv
  - RenderEnv access policy clarification (kernel direct, renderers via props)
  - $slot parameter access documentation
  - ValidationContext definition
- Created comparison analysis: `docs/experiments/next-gen-runtime-vs-current-flux-comparison-v5.md`
- Key findings from comparison:
  - Current Flux scores 8.5/10 vs v5's 6.9/10
  - Flux's seven-primitive model with promotion test is more mature
  - Flux's lexical root dependency tracking prevents over-invalidation
  - Flux's prototype-chain scope optimization is critical for performance
  - Flux's `useSyncExternalStore` integration solves real concurrent mode issues
  - v5's Effect algebra and draft form model could enhance Flux
- Verdict: Current Flux is the genuinely next-generation design; v5 offers theoretical elegance without practical proof
- Next step: Consider adopting v5's Effect algebra and draft form concepts to enhance current Flux
