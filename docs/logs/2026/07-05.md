  - **Core insight**: The current Flux `ApiResponse` type (`{ok, status, data}`) was invented without matching the nop-entropy standard (`{status, code, msg, data, errors}`). The backend's `isOk()` is `@JsonIgnore` computed from `status == 0`.
  - **Design**: `ok` becomes a computed property at the runtime normalization layer (`ok = status === 0`), mirroring backend `isOk()`. Fetcher returns raw backend JSON; runtime computes `ok` before responseAdaptor.
  - **Error handling**: `msg`/`code`/`errors` become first-class fields read from response top level, not guessed inside `data.message`/`data.msg`.
  - **`selection` parameter**: Added GraphQL-style field selection (`"id,name,role{id,label}"`) to `ApiSchema` and `ExecutableApiRequest`, corresponding to nop-entropy `FieldSelectionBean`.
