- **X2 可阻止事件（schema-driven preventDefault / stopPropagation）— 全 4 Phase 执行完成，Plan Status → completed。** Plan: `docs/plans/2026-06-21-2146-2-x2-preventable-events-plan.md`。把 Flux 事件系统从「runtime contract 已暴露 `preventDefault`/`stopPropagation`，但 author 无法在 schema 声明式请求 + timing hazard」收敛为「schema-level 声明 + 同步阻塞 native default + 既有命令式路径继续工作」。
- **Phase 1（Decision）**：4 条裁定落地。
  - 字段命名：`ActionShapeFields.preventDefault?: boolean | string` + `stopPropagation?: boolean | string`（与 DOM API 一致，与既有 `when` 同形；拒绝 amis `rendererEvent`）。
  - Timing 模型：sync prevention 发生在 `node-renderer-resolved.tsx` 的 per-event-key handler 内、`helpers.dispatch` 之前。求值顺序 normalize event → preventDefault expression → sync 调 native `event.preventDefault()` → stopPropagation expression → sync 调 native `event.stopPropagation()` → dispatch。`when:false ∧ preventDefault:true` 裁定：prevention 在 `when` 之前/独立求值生效，即 `when:false` 不 RUN action 但仍 sync 阻止 native default。

  - `stopImmediatePropagation` 不引入（roadmap scope 最小化）。
  - Docs：`docs/architecture/renderer-runtime.md` 新增 "Schema-Driven Prevention" 子节（含 timing contract、orthogonality with when、5 条 failure paths、out-of-scope）、`docs/components/form/design.md` 新增 "`preventEnterSubmit` 与 schema-driven `preventDefault` 的关系" 段落、`docs/references/naming-conventions.md` §4.4 X2 裁定行更新。
