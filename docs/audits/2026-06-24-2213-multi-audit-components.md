- **Files**: `packages/flux-renderers-data/src/crud-renderer.tsx:444-466,538`; declaration `packages/flux-renderers-data/src/crud-renderer-definition.ts:431-434`
- **Dimension**: 09
- **Evidence**:

  ```ts

  const base = { items: carrierRows, selectionMode: 'none', keyField, empty };
  if (listMode === 'list') {

  // :538 — remount-key workaround (with explicit "Follow-up" comment at :537)
  ```

- **Severity**: P2
- **现状 (Current)**: `item` and `card` are declared `kind: 'region'` so the compiler already extracts them into `props.regions.item.templateNode` / `props.regions.card.templateNode`. - **建议 (Recommendation)**: Invoke `props.regions.item.render({ bindings: { item, index } })` per item (as `list-renderer.tsx:87-95` does), or pass `props.templateNode.regions.item.templateNode` (already-compiled) to `helpers.render` so no recompilation occurs.
- **误报排除 (False-positive exclusion)**: `item`/`card` are schema-fragment regions carrying authored child schemas, not structural keys. On the live render path; v1 baseline forbids transitional allowances.
- **复核状态**: 独立复核通过 — 保留 (P2).
