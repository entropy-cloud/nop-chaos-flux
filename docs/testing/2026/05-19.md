## 2. Performance Table Full Stress 在 StrictMode host mutations 后出现 `$slot.record` 渲染错误

- **所属模块**: playground / performance-table / table row scope cache / scoped slot region / React StrictMode
- **问题描述**: 在 `#/performance-table` 页面切换到 `Full Stress` 后，执行 `Run 20 Host Mutations` 时，表格 cell 与 expanded row 中的 `${$slot.record.*}` / `${$slot.record.children}` 表达式可能大量报错；happy-dom 回归中 `console.error` 捕获到 312 次 render error。
- **预期行为**: Full Stress 模式在 StrictMode 下完成 host mutations 后仍应保留当前行 `$slot.record` 作用域，页面无 `console.error` / `pageerror`。
- **发现方式**: 回归验证 / 探索性测试
- **状态**: 已修复（`packages/flux-renderers-data/src/table-renderer.tsx` 的 table owner key 已纳入 `RendererRuntime.runtimeId`，避免 keyed `SchemaRenderer` remount 后复用旧 runtime 的 row-scope cache；新增 `data-table-row-scope-identity.test.tsx` keyed StrictMode remount 覆盖，并通过 performance-table happy-dom 与 Playwright focused 回归）
