# Manual Test Issues — 2026-05-26

## 7. input-tree 点击条目会意外切换整节点展开状态

- **所属模块**: playground / input-tree
- **问题描述**: 在 `http://localhost:5173/#/lab/input-tree` 页面中，点击某个具体条目本身时，会同时触发该节点的展开或收起，导致节点意外关闭或展开。这与常见 tree 交互不一致，用户难以稳定完成条目选择或聚焦。
- **预期行为**: 点击条目内容应只执行条目选中、聚焦或对应项点击行为；只有点击专用展开/收起控件时，才切换该节点的展开状态。
- **发现方式**: 人工测试
- **状态**: 已修复（`packages/flux-renderers-form-advanced/src/tree-controls.tsx` 为 `input-tree` 声明 `frameRootTag: 'div'`，避免外层 `<label>` 包装把节点文本点击转发到内部展开按钮；`src/__tests__/tree-structure.test.tsx` 新增点击父节点文本后仍保持展开的回归测试）

## 8. Report Designer 按钮区被拆成多行窄按钮排布

- **所属模块**: report-designer / spreadsheet-toolbar
- **问题描述**: Report Designer 顶部 control bar 当前会被拆成多行，每行只有 3 到 4 个按钮，导致工具栏高度明显膨胀，主工作区垂直空间被大量挤占。点击 sheet 单元格后还会在 toolbar 下方出现“输入单元格值”输入框，进一步把页面向下撑高；这个 toolbar cell editor 与网格内联编辑、右侧 property panel 之间职责混乱。
- **预期行为**: Report Designer 顶部 control bar 应优先保持为单行横向动作带；当按钮数量超出可用宽度时，应通过横向滚动等单行方案处理，而不是自动换成多排。Spreadsheet 的 Excel-like baseline 应为 selection-first：单击/聚焦只选中单元格，编辑通过双击、`Enter`、`F2` 或直接键入进入单元格内联编辑；toolbar 下方不应再渲染单独的 cell value editor。
- **发现方式**: 人工测试
- **状态**: 已修复（真实页面路径是 playground `ReportDesignerDemo` 使用的 `@nop-chaos/spreadsheet-renderers` `SpreadsheetToolbar`。现已将 live toolbar 样式改为单行 `nowrap` + 横向滚动，并为 toolbar groups/status 增加 `flex-shrink: 0`；同时移除了 toolbar 下方单独的 cell value editor，明确回到 Excel-like baseline：单击仅选中，编辑通过双击、`Enter`、`F2` 或直接键入进入单元格内联编辑。为避免编辑态把 22px 行高撑大，`packages/spreadsheet-renderers/src/spreadsheet-grid/inline-controls.tsx` 现使用文档化的 spreadsheet canvas 特例专用 input，并在 `canvas-styles.css` 中约束其尺寸严格贴合 cell 盒子。回归覆盖已补到 `packages/spreadsheet-renderers/src/__tests__/grid-selection.test.tsx`、`packages/spreadsheet-renderers/src/spreadsheet-toolbar.test.tsx`、`apps/playground/src/pages/report-designer-demo.test.tsx` 与 `tests/e2e/report-designer-demo.spec.ts`）
