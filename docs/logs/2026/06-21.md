- **showIcon / showOutline 裁定：删除字段。** 依据：当前 schema 未定义图标/引导线的数据来源（无 `iconField`/`iconKey`），实现需一整套图标解析设计，属 feature（E2d 范围或独立 feature plan）而非漂移修复。保留"声明了但设了无效"的字段正是 E0b 要修的漂移本身。依据 Flux 原则"不以 amis 为标尺"（analysis §0.2 / §5）。后续如需图标，须以独立 feature plan 提供完整图标来源设计后再加字段。落地：从 `InputTreeSchema` 移除 `showIcon` + `showOutline`；从 `TreeSelectSchema` 移除 `showIcon`（Phase 3 执行）。
- **cascade 最终语义固化（写入两份 design.md）**：
  - 仅多选（`treeMode: 'checkbox'`）生效；单选（`radio`/`normal`）不受影响。
  - 向下传播：点击父节点翻转全部"可选后代"（`onlyLeaf` 决定叶子还是全部）+ 父本身（`onlyLeaf: false` 时）。
  - 向上派生：父节点 `checked`/`indeterminate` 由可选后代选中态派生，非独立存储。
  - `onlyLeaf` 优先于 `cascade`：父值永不写入 value；cascade 仅向叶子传播。
- 更新 `docs/components/input-tree/design.md`、`docs/components/tree-select/design.md`：删除全部"漂移待 E0b"措辞；Flux 决策表 cascade 改"实现"、showIcon/showOutline 改"不采纳（删字段）"。
