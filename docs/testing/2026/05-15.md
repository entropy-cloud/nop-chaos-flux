# Manual Test Issues — 2026-05-15

## 11. Drawer 中 textarea/input 无法输入任何字符

- **所属模块**: flux-react / dialog-host / drawer (vaul)
- **问题描述**: 在 `http://localhost:5173/#/lab/drawer` 页面中，打开右侧 Drawer 后，表单内的 textarea 和 input 控件无法输入任何字符。点击控件可以聚焦，但键盘输入无效。
- **根因分析**: vaul 的 `DrawerPrimitive.Content` 在 `onPointerDown` 中调用 `onPress()`，执行 `setIsDragging(true)` 和 `event.target.setPointerCapture(event.pointerId)`。对于 left/right 方向的 drawer，`shouldDrag()` 始终返回 `true`，导致点击内容区域时 vaul 进入拖拽模式，pointer capture 抢占了 textarea 的输入事件流，浏览器无法正常派发键盘输入到 textarea。
- **修复方案**: 在 `DrawerView` 渲染 `<Drawer>` 时传入 `handleOnly` 属性，使 vaul 跳过内容区域的拖拽拦截，仅允许通过 drag handle 拖拽。关闭 drawer 仍可通过关闭按钮、Escape 键、点击遮罩层实现。
- **单元测试**: `packages/flux-react/src/__tests__/drawer-form-input.test.tsx` — 验证 openDrawer 打开的表单内 input 和 textarea 可以正常接受输入，且多次 change 事件后值正确保留。happy-dom 下全部通过（387/387）。
- **发现方式**: 人工测试
- **状态**: 已修复（`packages/flux-react/src/dialog-host.tsx` DrawerView 添加 `handleOnly` 属性）
