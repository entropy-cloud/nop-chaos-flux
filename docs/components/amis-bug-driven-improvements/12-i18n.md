# 12 i18n — Amis Bug-Driven Improvements

> Flux owner docs: `packages/flux-i18n/`, `docs/architecture/api-data-source.md` (dynamic schema), `docs/references/quick-reference.md`
> amis cluster: `i18n` (10 issues)
> Priority summary: Flux has a dedicated `flux-i18n` package (NOT amis's per-component baked strings). The strongest signal is that amis failed to localize **dynamically-loaded / remote schema** and **runtime-owned default strings** (notify titles) — flux's dedicated package is well-positioned but must guarantee these paths explicitly.

## A. Locale Resolution Coverage

| #   | Property                                                                                                                                                                                                            | Signal     | Severity | AMIS-REF                                                                                                               |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------- | -------- | ---------------------------------------------------------------------------------------------------------------------- |
| I1  | Locale resolution (both message-key lookup AND any schema-string i18n) applies uniformly to **static, schemaApi-fetched, and dynamically-injected schema**; the resolution pass re-runs on locale change.           | DESIGN-GAP | P0       | #9111, #9564, #4537                                                                                                    |
| I2  | Every user-facing default string (notify title/level fallbacks, default toast/error messages, default action labels) resolves through the i18n registry, never a literal — no baked-in strings bypass the registry. | DESIGN-GAP | P1       | #2078, #2422                                                                                                           |
| I3  | Message API supports (1) external resource bundles keyed by locale, (2) a `t(key, params)` expression/filter, (3) parameterized/interpolated messages.                                                              | DESIGN-GAP | P1       | #7140, #7035, #5557                                                                                                    |
| I4  | A locale change is a first-class reactive event: all rendered strings (static schema, dynamic schema, format filters) re-resolve in BOTH directions (zh→en→zh and en→zh→en) without remount.                        | TEST-GAP   | P1       | #10404                                                                                                                 |
| I5  | Locale-aware formatters (dates, numbers, relative time / `fromNow`) produce locale-correct output for each registered locale; flipping locale changes formatter output.                                             | TEST-GAP   | P2       | #6964 — **RESOLVED (B7)**: watch-only (locale-aware formatters delegate to flux-i18n/Intl construct-true; P2 low-risk) |

**Recommended actions:**

- I1: Add design note to `flux-i18n` / `api-data-source.md`: locale resolution MUST apply uniformly to static, schemaApi-fetched, and dynamically-injected schema, re-running on locale change.
- I2: Add design note: every user-facing default string resolves through the i18n registry; hard rule with enforcement path.
- I3: Add design note: confirm message API supports external bundles + `t(key, params)` expression + parameterized messages.

**Recommended tests:**

- I1: a remotely-fetched schema node with a localized key renders in the active locale and re-renders on locale switch.
- I2: under a non-default locale, a fetcher-error notify and a default toast render translated titles (guard against #2078 literal leak).
- I3: `${t('greeting', {name})}` resolves from an external bundle with parameter substitution in the active locale.
- I4: toggle locale zh→en→zh and en→zh→en → every localized surface re-renders correctly on each toggle without remount.
- I5: date/number/relative-time formatters produce locale-correct output; flip locale → output changes.

## Highest-Leverage Items

1. **I1** — locale resolution covers schemaApi/dynamically-loaded schema (THE amis i18n regression; flux will hit it as soon as remote schema loading lands).
2. **I2** — no baked-in runtime strings outside the registry (the #2078 "系统消息" leak).
3. **I4** — locale switch is reactive in both directions (amis only worked one way).
