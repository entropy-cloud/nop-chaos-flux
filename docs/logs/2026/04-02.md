### 2026-04-02 (session 2)

- Added `ScopeRef.merge(data)` method for shallow-merging a `Record<string, unknown>` into scope's own data with change detection.
- Key decision: `merge` performs per-key `Object.is` comparison before writing — skips `setSnapshot` call entirely when no values actually changed, avoiding unnecessary reference changes and downstream reactive re-renders.
- Empty merge data (`{}`) is treated as a no-op and returns immediately.
- Interface change: added `merge(data: Record<string, unknown>): void` to `ScopeRef` in `packages/flux-core/src/types/scope.ts:24`
- Implementation: `packages/flux-runtime/src/scope.ts:192-214` — checks each incoming key against current snapshot; only creates new snapshot object if at least one value differs.
- Updated `DataSourceRenderer` to use `scope.merge()` instead of raw `scope.store?.setSnapshot({...})`: `packages/flux-renderers-data/src/data-source-renderer.tsx:31`
- Added 10 tests covering: merge new keys, overwrite, no-op on unchanged, change detection, reference stability, child scope isolation, empty data: `packages/flux-runtime/src/__tests__/scope-merge.test.ts`
- Fixed all downstream mock ScopeRef objects across 4 test files (`flux-formula`, `flux-react`, `flux-runtime`) to include `merge` stub.
- All 148 runtime tests pass; build clean.
