### 2026-07-08 (prefer nearest-ancestor built-ins over componentId targeting)

- **Authoring principle enforced**: contextual built-ins (`submitForm`, `closeSurface`) resolve to the nearest enclosing owner already (`ctx.form` via `useCurrentForm()`; `ctx.dialogId`/`closeTop()`), so schema should use them instead of `component:<method>` + `componentId`/`componentName` whenever the target is the owning form/surface. Reserve `component:` targeting for genuine cross-tree targets (e.g. refreshing a sibling/ancestor CRUD from inside a dialog).

### 2026-07-08 (crud-demo refactor to idiomatic flux)

- **crud-demo.json rewritten to the most natural/concise form**, fully leveraging flux built-in capabilities (was `data-source` + `source` binding + `showToast` + separate `confirm` actions):
  - **`loadAction`** replaces `data-source` + `source` + `onRefresh→refreshSource`. The CRUD now owns its fetch (`loadAction: { action: 'ajax', args: { url, method } }`); `loadAllData: true` keeps the demo's small dataset client-side (pagination/filter/sort in-browser, no server query logic needed in the mock).
  - **`messages.success`** replaces every `then`+`showToast` chain (新增成功 / 更新成功 / 删除成功 / 批量删除成功) — ajax `messages` is the built-in toast-feedback capability.
  - **`confirmText`** on the delete ajax actions replaces the separate `confirm` action + `when: ${result.data.confirmed}` gate (single delete + bulk delete) — the built-in confirmText gate calls `env.confirm` directly.
  - Intro text + page seeding updated (`apps/playground/src/pages/crud-demo-page.tsx`: removed the now-unused `ds_users` pre-seed + `initialSeed`).
- **Fixed two CRUD `loadAction` bugs** exposed by making the demo use it end-to-end:
  1. **`$crud.total` not reactive after async load** — `loadAction` keeps rows/total in React state, but `createReadonlyScopeBinding`'s projected store only re-notifies on parent-scope writes (unlike the `source` path where data lives in scope). Added a private `__crudLoadRevision` scope bump in `crud-renderer.tsx` on each load-result change so `$crud` subscribers (footer `${$crud.total}`, statistics) re-read the latest summary. Diagnosed via `page.evaluate` (footer text), not screenshots.
  2. **`component:refresh` did not reload after mutation** — `useCrudLoadAction.reload()` called `loadReaction.force()`, which re-fires the reaction's ajax but whose result is never captured into rows/total (only the imperative `dispatch()` in the load effect calls `setRows`/`setTotal`). Replaced `force()` with a `reloadNonce` state in the effect deps so reload re-triggers the result-capturing dispatch.
