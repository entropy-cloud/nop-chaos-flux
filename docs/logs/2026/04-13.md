### 2026-04-13 (session 39)

- Fixed `detail-view` (and by extension any renderer using `parentForm.setValue`) not updating the viewer after the second confirm.
- Root cause (deeper than session 38's fix): `FormRuntime.setValue` and `setValues` in `packages/flux-runtime/src/form-runtime.ts` called `store.batchUpdate` without first calling `setLastChange`. `NodeRenderer` subscribes to `props.scope.store` and uses `store.getLastChange()` as its `getSnapshot`. This ensures every field write produces a fresh `lastChange` object, so `NodeRenderer`'s snapshot always changes and expression re-evaluation is triggered correctly.
