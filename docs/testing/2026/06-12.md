# Manual Test Issues — 2026-06-12

## 1. flux basic key-value demo submit wiring reports debugger failure without page feedback

- **所属模块**: playground / flux basic / key-value / form submit wiring / debugger
- **问题描述**: 在 `http://localhost:5173/#/flux-basic` 页面 `Key-value child cells` 卡片中，给 `Metadata cells` 新增一行空数据后点击 `Submit key-value demo`，实际提交失败。`nop-debugger` 中能看到提交失败事件，但页面上没有任何失败信息或字段错误展示。
- **预期行为**: 该按钮应走表单标准提交链路。若数据不合法，应在页面字段上显示对应校验错误；不应因为错误的组件能力 payload 导致只在 debugger 中出现 action failure。
- **发现方式**: 人工测试
- **状态**: 已修复（`apps/playground/src/pages/fluxBasicPageSchema.json` 改为把 `/api/composite-demo` 配置到表单 `submitAction`，按钮只触发 `component:submit`；`apps/playground/src/pages/flux-basic-page.debugger.test.tsx` 新增页面级回归，覆盖“新增空 metadata 行后 submit 会显示字段校验且不会误发 API 请求”）

## 2. condition-builder AND/OR 切换看起来有明显延迟

- **所属模块**: `condition-builder` / conjunction toggle / interaction feedback
- **问题描述**: 人工测试中，`condition-builder` 的 AND/OR 切换点击后看起来要延迟一段时间，选中态才切换，容易误判为状态链路慢或 store 更新滞后。
- **预期行为**: 点击后选中态应立即切换，避免视觉过渡造成“慢响应”感知。
- **发现方式**: 人工测试
- **诊断结论**: 新增 `packages/flux-renderers-form-advanced/src/condition-builder/condition-builder-latency.test.tsx` 后确认逻辑状态切换和 form store commit 都是有界的，没有发现明显逻辑迟滞；问题主要来自 conjunction pill 自身的 `transition-colors` 视觉过渡。
- **状态**: 已修复（`packages/flux-renderers-form-advanced/src/condition-builder/condition-group.tsx` 将 AND/OR pill 从 `transition-colors` 调整为 `transition-none`；新增测试校验 pressed state 正常切换、一次切换 commit 有界、按钮不再带颜色过渡类）
