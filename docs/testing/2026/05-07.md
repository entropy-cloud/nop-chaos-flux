# Manual Test Issues — 2026-05-07

## 1. Performance Table 中 tag-list 行级校验错误被共享到多行

- **所属模块**: playground / performance-table
- **问题描述**: 在 `/#/performance-table` 的表格中编辑 `Tags` 列时，选择或取消 tag 后会出现 `...requires at least one tag` 报错，看起来像所有行都被污染。最终定位确认并不是 row-level validation 广播，而是 `packages/flux-renderers-form-advanced/src/tag-list.tsx` 无论 schema 是否声明 `required`，都会无条件注册“至少选择一个 tag”的 runtime validation；而 `performance-table` 的 tag 列本来就不是必填。
- **预期行为**: 非必填 `tag-list` 只做值切换，不显示 required 错误；只有声明 `required: true` 时才校验至少选择一个 tag。
- **发现方式**: 人工测试 + 单元测试 + e2e 回归
- **状态**: 已修复（`packages/flux-renderers-form-advanced/src/tag-list.tsx` 仅在 `required` 时注册非空校验；`packages/flux-renderers-form-advanced/src/{tag-list.test.tsx,__tests__/form-runtime-fields.test.tsx}` 与 `tests/e2e/performance-table.spec.ts` 补充回归覆盖）
