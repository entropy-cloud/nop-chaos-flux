  - Routing: new `RouteSpec` kinds `showcase` / `showcase-page` (`apps/playground/src/route-model.ts`), wired in `App.tsx` + `home-page.tsx`; `#/complex-pages[/<id>]` deep links. `route-matrix.test.ts` extended (34 tests).
  - Shell + model: `apps/playground/src/complex-pages/complex-pages-showcase.tsx`, `complex-pages-model.ts` (4 categories × 9 pages), `complex-pages-registry.ts`.
- **9 demo pages** (`complex-pages/pages/`), each schema-driven via a shared `SchemaRenderer`:
  - `standard-crud` (reuses `schemas/crud-demo.json`), `tree-crud`, `inline-edit-table`, `advanced-query`, `master-detail`, `detail-subtables`, `form-wizard`, `complex-form`, `dashboard`.
- **Shared infra** (`complex-pages/shared/`): `mock-backend.ts` (in-memory depts/users/orders/items/logs/addresses/payments/shipments/budgets), `showcase-env.ts` (single fetcher handling all `/r/...` endpoints + notify/confirm/loadDict), `confirm-bridge.tsx`, `render-host.tsx`, `page-frame.tsx`.
- **Key flux patterns confirmed/used** (recorded for future demos):
  - CRUD `loadAction` requires explicit `dependsOn` for external scope vars, BUT an external `dependsOn` change does NOT re-capture rows when `loadAllData:true` (force() fires ajax, result not captured). For external-filter linkage, route through a `data-source` with `dependsOn` + `crud source='${ds}'` instead (see `tree-crud`). Query-form filters work via loadAction because query state IS in the capture deps.
  - `data-source` `sendOn` is a RAW expression (compiler auto-wraps in `${}`) — never write `sendOn: '${x}'`; use `sendOn: 'x'`.
  - `data-source` default publish = unwrapped `response.data`; no `resultMapping` needed to expose `.items`.
  - `quickSaveItemAction` is `kind:'prop'` → its `args.data` expressions are eagerly evaluated in CRUD scope (no `record`); use `includeScope:'*'` + read `body.record` server-side. Real per-row number inputs need `quickEdit.body` (`{type:'input-number',name:'record.q',frameWrap:false}`); `quickEdit:{type:...}` is ignored (always text input).
  - Table cell text binds row as `record` (NOT `$slot.record`); `$slot.record` only in operation-column buttons.
  - flux formula is Excel-like — JS `.map()`/arrow fns are invalid in `${}`; build option lists server-side (dedicated `*__pickerOptions` endpoint returning `{label,value}[]`).
  - `form valuesPath` publishes values to scope on every change (master/detail selection via radio-group/select inside such a form reliably drives page-scope state).
- **Review process**: two independent fresh-context sub-agents audited (1) data-flow/runtime correctness and (2) UX/conventions. Data-flow audit caught 5 latent bugs the smoke tests didn't cover (tree empty + filter no-op, `${}`-wrapped sendOn, `?.[]` unsupported, row expr in eager prop) — all fixed by switching to the patterns above.
- **Verification**: playground typecheck ✅ · lint ✅ · build ✅ · 105/105 unit tests pass (incl. new `complex-pages.test.tsx` with 11 runtime smoke tests asserting real data flow per page, e.g. tree-select filters table, master-detail selection loads all sub-tables).

### 2026-07-09 (playground: Complex Pages — 远程标签页 + 动态加载分析)

**问题**：用户问是否需要给 tabs 加 `schemaApi` 实现"不点击不加载"的 tab 动态内容加载。

**分析结论（经过完整源码验证）**：

- **amis tabs 没有 `schemaApi`**（已验证 `~/sources/amis` 的 `TabSchema` 和 `AMISTabsSchema`，无 `api`/`schemaApi`/`deferApi`/`loadData` 字段）。amis 的懒加载靠 `mountOnEnter: true`（默认）+ tab body 内嵌有 `api` 的子组件（`service`/`crud`/`chart`）。
- **Flux tabs 已经具备完全等价的能力**：`mountOnEnter: true` 已实现（`tabs.tsx` `resolveTabKeepMounted()` + `activated` Set），并有 3 层测试覆盖（单元测试 `layout-family-enhancements.test.tsx:284` + `tabs-owner-lifecycle.test.tsx:20`，e2e 测试 `layout-family-enhancements.spec.ts:122`）。设计文档 `docs/components/tabs/design.md` §2 已记录。
- **不需要给 tabs 加任何 `schemaApi`**。tab body 内嵌 `DynamicRenderer`（`type: 'dynamic-renderer'`）即可实现：`mountOnEnter: true` 阻止 tab body DOM 挂载 → DynamicRenderer 不初始化 → 不触发 `loadAction` → 点击 tab 后 body 挂载 → DynamicRenderer mount → `loadAction` 自动执行 → 远程 schema 加载渲染。这是**组件组合而非 tabs 自身能力**，零改动 tabs renderer。

**新增"远程标签页"演示（`dynamic-tabs-page.tsx`）**：第 14 个 Complex Page（`data-lists` 分类）。

- Tab "概览"：预定义静态 card schema
- Tab "远程数据"：`mountOnEnter: true`，body 内嵌 `dynamic-renderer`，`loadAction` 通过 `action:'ajax'` 从 `/r/RemoteTabContent` 获取 CRUD schema（`source:'${remoteTabItems}'`，数据由 `ShowcaseSchemaHost` `data` prop 注入 page runtime scope）
- 测试断言：远程 tab body 初始 `queryByTestId → null`，点击后 CRUD 渲染含"数据项 A"和"100"
- **验证**：typecheck ✅ · lint ✅ · build ✅ · **110/110 单测**（新增 1 个 smoke 测试）。
- **Not committed** — awaiting explicit user request.
