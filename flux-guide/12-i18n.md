# 国际化（i18n）

> Flux 自带完整 i18n 系统，但 **API 在独立包 `@nop-chaos/flux-i18n`**，不从 `@nop-chaos/flux-react` 或 `@nop-chaos/flux` 入口导出。底层用 `i18next` + `react-i18next`（peerDependencies，宿主须安装）。

---

## 核心 API

全部从 `@nop-chaos/flux-i18n` 导入（`SupportedLanguage = 'zh-CN' | 'en-US'`，见 `packages/flux-i18n/src/i18n.ts:7`）：

| API                          | 签名                                        | 用途                                                          |
| ---------------------------- | ------------------------------------------- | ------------------------------------------------------------- |
| `initFluxI18n(options?)`     | `(opts?: FluxI18nOptions) => i18n`          | 单例初始化，默认 zh-CN。在 `ReactDOM.createRoot` 之前调用一次 |
| `resetFluxI18n()`            | `() => void`                                | 重置回 identity formatter（测试隔离用）                       |
| `t(key, options?)`           | `(key: string, opts?) => string`            | 函数式翻译，**生产代码主用此 API**                            |
| `useFluxTranslation()`       | `() => { t, i18n }`                         | React Hook，语言切换时触发重渲染                              |
| `changeLanguage(lng)`        | `(lng: SupportedLanguage) => Promise<void>` | 切换语言（仅支持 `zh-CN` / `en-US`）                          |
| `getCurrentLanguage()`       | `() => SupportedLanguage`                   | 读当前语言                                                    |
| `addResources(lng, ns, res)` | `(lng: SupportedLanguage, ns, res) => void` | 增量合并资源（走 `i18next.addResourceBundle`）                |

`FluxI18nOptions`（`packages/flux-i18n/src/i18n.ts:50-56`）：`{ lng?: SupportedLanguage; fallbackLng?: SupportedLanguage; resources?: Resource; debug?: boolean; react?: boolean }`。`react` 字段控制是否 `use(initReactI18next)`（默认 `true`）。默认 namespace 为 `flux`。

## schema 内使用 `${t('key')}`

公式层注册了 `t` builtin（`packages/flux-formula/src/builtins.ts`），任何 `SchemaTpl` / `SchemaExpression` 字段可直接用：

```json
{ "type": "text", "text": "${t('flux.common.save')}" }
{ "type": "button", "label": "${t('flux.common.confirm')}" }
```

> key 会被归一化：缺前缀时自动补 `flux.`；动态 key（如 `${'flux.operator.' + op}`）允许，但 lint 工具要求把动态前缀清单显式声明到 `scripts/check-i18n-keys.mjs` 的 `DYNAMIC_KEY_MAPS`。

---

## 资源加载：静态 import + addResources，**无异步 loader**

flux **不提供** `loadLocale` / `loadMessages` / `loadResource` API。两种加载方式：

### 1. 默认 bundle（构建期内置）

`@nop-chaos/flux-i18n` 自带 `zh-CN.ts`（约 607 个 key、686 行）+ `en-US.ts`（约 605 个 key、688 行），`initFluxI18n()` 调用即生效，无需额外 import。

### 2. 宿主自定义资源

```ts
import { initFluxI18n, addResources } from '@nop-chaos/flux-i18n';
import enShared from './locales/en/shared.json';

initFluxI18n(); // 默认 zh-CN
addResources('en-US', 'flux', enShared); // 增量合并到 en-US
```

子路径导出（`packages/flux-i18n/package.json`）：`@nop-chaos/flux-i18n/locales/zh-CN`、`@nop-chaos/flux-i18n/locales/en-US`，可宿主端 `import` 后覆盖。

---

## `env.locale` 的角色：**声明性字段，flux 不消费**

`env.locale?: string` 在 `RendererEnv` 上声明，但 **flux 源码从不读取它**。它仅供宿主在自己实现的 `loadDict` / `loadPage` 缓存键里使用。flux 自身的语言切换走 `changeLanguage()` 或 `addResources()`，与 `env.locale` 无关。

---

## 渲染器内部硬编码文案

flux 所有渲染器（form / data / content / mobile / debugger / report-designer / word-editor 等）的内部文案（如 `flux.common.validating`、`flux.form.required`、`flux.table.empty`）都通过 `t('flux.xxx')` 调用。共 207 个文件 import `@nop-chaos/flux-i18n`，其中约 126 个直接 import `t` 函数，资源键已在 `zh-CN.ts` / `en-US.ts` 里定义。

`scripts/check-i18n-keys.mjs` 在 `pnpm lint` / `pnpm check` 时强制扫描：所有字面量 `t('flux.*')` 调用的 key 必须在 locale bundle 中存在，否则失败。

## 宿主集成步骤

```tsx
import { initFluxI18n, changeLanguage } from '@nop-chaos/flux-i18n';

// 1. 应用启动时初始化（默认 zh-CN）
initFluxI18n();

// 2. 切换语言
await changeLanguage('en-US');
```

```tsx
// 3. 宿主自己的组件需要翻译时
import { useFluxTranslation } from '@nop-chaos/flux-i18n';

function HostHeader() {
  const { t } = useFluxTranslation();
  return <h1>{t('flux.common.save')}</h1>;
}
```

## 重要限制

| 限制                                             | 说明                                                                                               | 影响                                                                                                                                                                |
| ------------------------------------------------ | -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`t()` 函数不触发重渲染**                       | flux 所有渲染器用 `t()` 函数式 API（约 126 处直接 import `t`），**未用** `useFluxTranslation` Hook | 调用 `changeLanguage()` 后，已挂载组件的翻译输出不会自动刷新；只有用 `useFluxTranslation` 的组件才会重渲染。如需整页切换语言后刷新，宿主需要 remount 子树或强制更新 |
| **语言仅支持 zh-CN / en-US**                     | `changeLanguage` / `addResources` 第一个参数类型为 `SupportedLanguage = 'zh-CN' \| 'en-US'`        | 其他语言需要扩展 `SupportedLanguage` 类型并自行准备 locale bundle                                                                                                   |
| **无异步资源加载 API**                           | 必须 `import` 静态资源 + `addResources`                                                            | 不能按需懒加载 locale，须在构建时打包或在运行时 fetch + `addResources`                                                                                              |
| **无 React Context Provider**                    | 仅靠 i18next singleton + `useTranslation(NS, { i18n })`                                            | 不存在 `<I18nextProvider>`，无法在同一页挂载两个独立 i18n 实例                                                                                                      |
| **flux-react / flux-bundle 入口不导出 i18n API** | 必须直接 `import from '@nop-chaos/flux-i18n'`                                                      | 宿主需把 `@nop-chaos/flux-i18n` 加入依赖                                                                                                                            |
| **schema 字符串无运行时翻译 pass**               | flux runtime 不对 schema 字面量做 i18n 替换                                                        | 所有需要在 schema 里翻译的字符串必须显式用 `${t('key')}` 表达式                                                                                                     |

## 内部架构（给贡献 flux 本身的人）

- **`t()` sink**：`flux-core/src/i18n-sink.ts` 提供 `setMessageFormatter` / `getMessageFormatter` 单例。flux-formula 的 `t` builtin 与 flux-runtime 的校验消息构造都从 sink 读，避免公式层直接依赖 i18next。
- **sink 写入方**：`initFluxI18n` 在初始化时把 `(key, params) => instance.t(normalizeTranslationKey(key), params)` 写入 sink。
- **ui 包桥接**：`@nop-chaos/ui` 不依赖 i18next，自己有最小英文 map + `setI18nGetter()` 桥接器（全局 `Symbol.for('nop.ui.i18nBridge')`）。`initFluxI18n` 时通过 `bindUiI18n` 把 ui 包的 `t()` 路由到 flux-i18n 实例。
- **lint 强约束**：`scripts/check-i18n-keys.mjs` 扫描 `apps/`、`packages/` 下所有字面量 `t('flux.*')` 调用，强制 key 必须在 locale bundle 中存在。
