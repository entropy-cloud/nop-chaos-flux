# Development Log — 2026-04-13

### 2026-04-13 (session 1)

- Authored Flow Designer tree mode design document and two domain-specific example files.
- Links: `docs/architecture/flow-designer/tree-mode.md`, `docs/examples/dingtalk-workflow-tree.md`, `docs/examples/action-flow-tree.md`
- Key decisions:
  - Tree DSL has exactly 3 structural primitives: `child` (chain sequence), `branches` (fan-out), `TreeNodeBranch.child` (branch subtree). No `BranchMode` enum — routing semantics are domain data in `data`.
  - `then` / `onError` in action schema are condition-branch fan-outs, identical in structure to DingTalk's condition gateway. No extension needed to the tree model.
  - Tree mode and graph mode share the same React Flow canvas. Tree projects to flat `GraphNode[] + GraphEdge[]` via `TreeProjection`.
  - `TreeNodeTypeConfig` extends existing `NodeTypeConfig` with optional `tree` constraints (`allowBranches`, `allowChild`, `isTerminal`). All rendering and editing capabilities (body, inspector, appearance) are reused unchanged.
- Analyzed:
  - Existing `GraphDocument`, `NodeTypeConfig`, `EdgeTypeConfig` types in `packages/flow-designer-core/src/types.ts`
  - Action algebra spec (`docs/architecture/action-algebra-formal-spec.md`) and action graph authoring (`docs/architecture/action-graph-authoring.md`)
- Updated: `docs/index.md` — added tree-mode entry in "Read This First" table, example entries, and active source of truth list
